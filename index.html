Overview

What is LoRaWatch?

Server rooms are the backbone of modern IT infrastructure, yet they remain vulnerable to environmental risks — overheating, humidity, toxic gas buildup — and physical security threats such as unauthorized access.

LoRaWatch is a wireless sensor network built around the ESP32-S3 microcontroller that continuously monitors six environmental and access parameters. All sensor readings are packed into a compact 16-byte payload, encrypted with AES-128-ECB, and transmitted wirelessly over LoRa radio to a base station.

A Python Flask server receives, decrypts, and stores every reading in a local database, then pushes live updates to an authenticated web dashboard via WebSocket. An on-board AI risk engine scores each reading from 0 to 100 and categorises the environment as Low, Medium, High, or Critical risk in real time.

Capabilities

Key Features

Dual Temperature Monitoring

Two DHT11 sensors independently track server equipment temperature and ambient room temperature, triggering alerts when either crosses safe thresholds.

Humidity & CO₂ Sensing

Continuous monitoring of room humidity (alert above 70%) and CO₂ concentration (alert above 1000 ppm) to prevent condensation and air quality hazards.

NFC Access Control

PN532 NFC reader authenticates ISO 14443-A cards and tags. Each scan is logged with timestamp, person name, and access result — granted or denied.

Motion Detection

PIR sensor detects human presence. Motion events without valid NFC authorization trigger immediate security alerts on the dashboard.

Door State Monitoring

Magnetic reed switch continuously monitors whether the server room door is open or closed, logging every state change with a timestamp.

AES-128 Encryption

Every payload is encrypted with AES-128-ECB using a pre-shared key hardcoded at firmware build time. Encryption takes under 1 millisecond on the ESP32.

LoRa Wireless Link

EBYTE E32-433T20D modules transmit 40-byte frames with CRC-16 Modbus integrity checking, achieving 95%+ packet delivery through walls and interference.

Live Dashboard

Authenticated Flask web app with persistent WebSocket connection. All panels update within 3 seconds of any event — no page refresh needed.

Multi-Channel Alerts

Alerts delivered simultaneously via dashboard popup notification, LCD screen on the transmitter node, and a buzzer with distinct alert tones.

System Design

Three-Layer Architecture

Layer 1

Sensor / Perception Layer

The ESP32-S3 transmitter node collects readings from all sensors every 1.5 seconds, formats them into a 16-byte binary payload, encrypts with AES-128-ECB, appends a CRC-16 checksum, and sends a 40-byte frame over UART to the LoRa radio module.

Layer 2

Communication Layer

EBYTE E32-433T20D LoRa modules operate in transparent UART mode. The receiver node parses incoming frames using a three-state UART parser, verifies the CRC-16 checksum, and forwards validated payloads to the server via HTTP POST.

433 MHz LoRaCRC-16 Modbus AES-128-ECBHTTP POST
WebSocket / HTTP
Layer 3

Application / Presentation Layer

The Flask server decrypts the payload, reconstructs all sensor readings, runs the AI risk scoring engine, stores everything in SQLite, and instantly pushes a WebSocket event to all connected dashboard browsers.

Components

Hardware Components

ESP32-S3

Dual-core Xtensa LX7 @ 240 MHz, 512 KB SRAM. Hardware-accelerated AES via mbedTLS. Used as both transmitter and receiver nodes.

GPIO · I2C · UART · mbedTLS

EBYTE E32-433T20D

Semtech SX1276 LoRa chip. 433 MHz band, transparent UART mode. Reliable indoor packet delivery through concrete walls.

433 MHz · 20 dBm · SX1276

DHT11 (× 2)

Digital temperature and humidity sensor. ±2°C accuracy, ±5% RH. One monitors server equipment, one monitors ambient room conditions.

±2°C · ±5% RH · 3s interval

PN532 NFC Reader

Reads ISO 14443-A cards, tags, and stickers. Connected via I2C (SDA=GPIO21, SCL=GPIO0). Passive detection mode, 50 ms scan timeout.

I2C · ISO 14443-A · GPIO21/0

PIR Motion Sensor

Detects infrared radiation changes indicating human presence. Software debounce eliminates false triggers from HVAC airflow.

GPIO36 · Digital · Debounced

CO₂ Sensor

Analog output mapped via 12-bit ADC (0–4095 → 400–2000 ppm). Alerts triggered above 1000 ppm. Requires 3–5 min warm-up after power-on.

GPIO35 · 12-bit ADC · 400–2000 ppm

Magnetic Reed Switch

LOW when door closed (magnet present), HIGH when door opens. Directly monitored on a digital GPIO — no analog conversion needed.

GPIO39 · Digital · Active-HIGH

LCD Screen

Displays live temperature, humidity, CO₂, door state, and NFC scan results directly on the transmitter node for local visibility.

I2C · 16×2 · Real-time
Intelligence

AI Risk Scoring Engine

Every sensor reading is passed through a weighted multi-factor risk engine that produces a composite score between 0 (fully safe) and 100 (critical threat). The score is updated on every transmission cycle — approximately every 1.5 seconds.

The engine uses a trained MLPClassifier neural network combined with a deterministic rule-based fallback. The rule-based score always governs the final risk bucket, ensuring predictable and auditable behaviour even in edge cases.

Quick Start

# 1. Clone the repository

git clone https://github.com/ShahadB6/lorawatch.git

# 2. Install Python dependencies

pip install flask flask-socketio eventlet scikit-learn

# 3. Configure WiFi in receiver firmware

Open esp32_receiver/esp32_receiver.ino
Set WIFI_SSID to your network name
Set WIFI_PASSWORD to your network password
Set SERVER_URL to your PC's local IP (e.g. http://192.168.x.x:5000/api/data)

# 4. Run the server

cd lorawatch/server
python app.py

# 5. Open the dashboard

http://localhost:5000

Requirements

  • Arduino IDE with ESP32 board support
  • Python 3.8 or newer
  • EBYTE E32-433T20D LoRa modules
  • ESP32-S3 development boards (× 2)
  • All sensors listed in Hardware section
